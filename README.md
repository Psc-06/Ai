# BankShield Desktop

BankShield is a Windows desktop network vulnerability scanner built from three local components:

- `frontend/`: React + Vite + TailwindCSS dashboard rendered inside Electron
- `backend/`: Flask API that runs Nmap-based scans, enriches results with CVEs, stores data in local MongoDB, and generates PDF reports
- `electron/`: desktop shell that starts the backend automatically, loads the frontend, and packages the application into a Windows installer

## Architecture

At runtime the application stays on the local machine:

1. Electron launches the desktop window.
2. Electron starts the Flask backend in the background.
3. The React dashboard loads inside the Electron window.
4. The UI talks to Flask over `http://127.0.0.1:5000/api`.
5. Scan data and OWASP findings are stored in local MongoDB.
6. PDF reports are generated by Flask and saved locally through Electron's save dialog.

No application data is sent to external services.

## Prerequisites

- Windows with PowerShell
- Node.js 20+
- Python 3.11+
- MongoDB running locally on `mongodb://localhost:27017/`
- Nmap installed and available in `PATH`

## Install Dependencies

### Backend

```powershell
cd backend
.\venve\Scripts\Activate.ps1
pip install -r requirements.txt
pip install pyinstaller
```

### Frontend

```powershell
cd frontend
npm install
```

### Electron

```powershell
cd electron
npm install
```

## Development Mode

Run the desktop app with hot-reloading UI:

```powershell
cd electron
npm run dev
```

What happens in development mode:

- Vite starts the React development server on `http://localhost:5173`
- Electron launches the desktop shell
- Electron automatically starts the Flask backend from `backend/venve/Scripts/python.exe`

## Build the Production Desktop App

From the `electron/` directory:

```powershell
cd electron
npm run build
```

This build does the following:

1. Builds the React frontend into `frontend/dist`
2. Bundles the Flask backend with PyInstaller into `backend/dist/app/app.exe`
3. Packages Electron with `electron-builder`
4. Produces a Windows NSIS installer in `electron/dist`

## Output Files

- Backend bundle: `backend/dist/app/app.exe`
- Desktop installer: `electron/dist/`
- App icon: `electron/assets/icon.ico`

## Runtime Notes

- Scans require local network access and a working Nmap installation.
- Reports are exported as PDF files and saved locally.
- MongoDB is used as the local persistence layer for scan results and OWASP findings.
